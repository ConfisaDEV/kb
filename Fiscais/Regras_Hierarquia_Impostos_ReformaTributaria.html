Depois de saber o tipo de operação, o sistema procura as classificações nesta ordem:

Hierarquia de busca das classificações:

 1 - Tipo de Operação  + ITEM (Regra específica)

          Ex.: existe uma regra cadastrada dizendo que o produto 'Arroz 5kg' no tipo de operação 'Venda Interna' tem redução/diferimento específico de 5%.  

          -> A classificação dessa regra será aplicada.

 2 - Tipo de Operação + NCM (Regra específica)

          Ex.: Caso a NCM do item informado na nota tenha uma regra cadastrada dizendo Todos os produtos do NCM 1006.30.21 (arroz semi-branqueado) tem redução/diferimento específico de 2.5%.  

          -> A classificação dessa regra será aplicada.

 3 - Tipo de Operação (Padrão)

         Se não encontrar regras especificas nem para o Item e nem para a NCM.  

         -> A classificação padrão do tipo de operaçao será aplicada.

 

2- BUSCA DOS IMPOSTOS (%Aliquota, %Redução %Diferimento para os: CBS, IBSUF e IBSMUN)

Depois que temos o Tipo de Operação e as Classificações (CST e CClasTrib), o sistema vai buscar as porcentagens de impostos.

Importante: No cadastro de regras, cada imposto possui um campo Específico.

Se este campo estiver marcado, o sistema vai usar o valor informado na regra.

Se não estiver marcado, o sistema ignora a regra e continua procurando nos próximos níveis da hierarquia.

 

Todos os impostos seguem a mesma hierarquia, porém a %Redução tem uma etapa a mais.

 

Hierarquia de busca da %Alíquota e %Diferimento:

 1 - Tipo de Operação  + ITEM (Regra específica)

         Ex.: existe uma regra cadastrada dizendo que o produto 'Arroz 5kg' no tipo de operação 'Venda Interna' tem alíquota/diferimento de 2%.  

            -> Se o campo Específico estiver marcado, esse valor será aplicado.

 2 - Tipo de Operação + NCM (Regra específica)

         Ex.: existe uma regra dizendo que todos os produtos do NCM 1006.30.21 (arroz semi-branqueado) têm alíquota/diferimento de 1,8%.

            -> Se o campo Específico estiver marcado, esse valor será aplicado.

 3 - Tipo de Operação (Padrão Sefaz)

         Se não encontrar regra em nenhum dos níveis anteriores, o sistema vai aplicar o valor padrão definido pela Sefaz.

 

Hierarquia de busca da %Redução:

 1 - Tipo de Operação  + ITEM (Regra específica)

         -> Se o campo Específico estiver marcado, esse valor será aplicado.

 2 - Tipo de Operação + NCM (Regra específica)

         -> Se o campo Específico estiver marcado, esse valor será aplicado.

 3 - Tipo de Operação + Cadastro de Classificação Tributária do IBS/CBS (Regra específica)

      Se não houver regra marcada como específica, o sistema vai buscar o valor de redução diretamente na tabela oficial importada do Excel da Sefaz (CAD0229)

 4 - Tipo de Operação (Padrão Sefaz)
